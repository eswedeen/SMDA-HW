Spotify Audio Analysis

Welcome to the Spotify Audio Analysis project

Erik

Introduction

The purpose of this project was to experiment with using web APIs and Python code to perform an analysis with a data set that was retreived using web API calls. for this project, we wanted to indentify characteristics of songs that were indicative of popularity for a given genre.

Spotify has some useful features built into their web APIs that attribute an index value (0 to 1) for various audio features (properties) of a given track. These properties attempt to quantify emotional and perceptual components to a song ranking it on positivity, intensity, likeliness of being performed live, likeliness of being a speech based track, etc.. Not all available features were included in the analysis.

A lightweight Python library, "Spotipy", was used to perform Web API calls.

For the purpose of this project, a collection of songs was sampled and analyzed statistically for each of the following features:

  • Energy: Perceptual measure of intensity, loudness, or noisiness.
  • Danceability: How suitable a track is for dancing.
  • Speechiness: Representation of how much speech is in a track.
  • Acousticness: Confidence measure for whether the track is acoustic or not.
  • Valence: Perceptual measure of positivity.

Procedure

To Determine what type of audio features would be indicative of success for a track within a popular genre, a sample of tracks would need to be collected based on genre. Since Spotift has a collection of playlists already populated categorically by genre, and the tracks within them selected for their popularity, this was chosen as a sample.

Several Spotify web APIs were utilized to identify these popular playlists, tracks, and audio features. The procedure for retreiving each of these is a follows:

  1. User Playlists: Retreives a specified user's public playlists. (In our case this is the user "spotify"
  2. Playlists_tracks: Retreives a selected playlist's tracks and track IDs (Based on retreived playlist ID.)
  3. Track Audio Features: Retreives a selected track's audio features (Based on retreived track ID)

Once each of these API calls had been made. The data was collected, stored, and assimilated into a DataFrame using the Pandas Library. And then analyzed using the Numpy and Scipy libraries. Once this was done, Matplotlib was use to plot Normal Distribution curves for the audio properties associated with each playlist of a given genre.
